# Ical-parser Specification

> Generated by openlore v1.0.0 on 2026-06-18 09:21

## Purpose

Parses meeting data from two distinct calendar text formats — iCal/ICS files and Outlook for Mac
plain-text clipboard output — into a normalized MeetingEvent structure, and sanitizes meeting
descriptions by stripping Microsoft Teams boilerplate content.

## Requirements

> `src/ts/icalParser.ts`

### Requirement: ParseIcs

The system SHALL parses a raw ics/ical string using the ical.js library into a normalized meetingevent.

> Implementation: `parseIcs` in `src/ts/icalParser.ts` · confidence: llm extracts title, date, start/end times, organizer, attendees (excluding organizer), description, location, and meeting url. meeting url is resolved from a teams-specific x-property first, then falls back to scanning description and location text.

#### Scenario: ValidICSWithFullEventData
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A raw ICS string containing a VEVENT with summary, dtstart, dtend, organizer, attendees,
description, and location
- **WHEN** parseIcs is called
- **THEN** A MeetingEvent is returned with all fields populated, organizer excluded from attendees list, and
Teams boilerplate stripped from description

#### Scenario: ICSWithTeamsMeetingURLInXProperty
- **GIVEN** A VEVENT containing the x-microsoft-skypeteamsmeetingurl property
- **WHEN** parseIcs is called
- **THEN** meetingUrl is set from the X-property value without scanning description or location

#### Scenario: ICSWithMeetingURLInDescriptionOnly
- **GIVEN** A VEVENT with no x-microsoft-skypeteamsmeetingurl but a Teams/Zoom/Meet link embedded in the
description
- **WHEN** parseIcs is called
- **THEN** meetingUrl is extracted from the description text

#### Scenario: ICSWithMeetingURLInLocationOnly
- **GIVEN** A VEVENT with no URL in the X-property or description but a meeting URL in the location field
- **WHEN** parseIcs is called
- **THEN** meetingUrl is extracted from the location text

#### Scenario: ICSWithNoMeetingURLAnywhere
- **GIVEN** A VEVENT with no meeting URL in X-property, description, or location
- **WHEN** parseIcs is called
- **THEN** meetingUrl is set to empty string

#### Scenario: ICSWithMissingOptionalFields
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A VEVENT that has only required fields and omits organizer, attendees, description, location
- **WHEN** parseIcs is called
- **THEN** MeetingEvent is returned with empty strings for missing fields and empty array for attendees

#### Scenario: EmptyICSInput
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** An empty or whitespace-only string is passed as raw
- **WHEN** parseIcs is called
- **THEN** An error is thrown with message 'ICS file is empty.'

#### Scenario: ICSWithNoVEVENT
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A valid ICS string that contains no VEVENT component
- **WHEN** parseIcs is called
- **THEN** An error is thrown with message 'No VEVENT found in the iCal file.'

#### Scenario: AttendeeMatchingOrganizerEmailIsExcluded
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A VEVENT where the organizer also appears in the attendee list
- **WHEN** parseIcs is called
- **THEN** The organizer is not duplicated in the attendees array

#### Scenario: OrganizerHasCNParameter
- **GIVEN** A VEVENT where the organizer property has a CN (common name) parameter
- **WHEN** parseIcs is called
- **THEN** organizer is set to the CN display name rather than the raw mailto address

### Requirement: ParseOutlookText

The system SHALL parses the plain-text meeting summary that outlook for mac places in the clipboard when dragging a calendar event.

> Implementation: `parseOutlookText` in `src/ts/icalParser.ts` · confidence: llm extracts title (lines before 'when:'), date/time, location (where:), organizer (note: line), attendees (invitees: line), and free-form description. returns null if the text does not appear to be an outlook meeting summary.

#### Scenario: ValidOutlookMeetingTextWithAllFields
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A plain-text string with When:, Where:, Note:, and Invitees: lines and a free-form description
- **WHEN** parseOutlookText is called
- **THEN** A fully populated MeetingEvent is returned with title, date, startTime, endTime, organizer,
attendees, description, location, and meetingUrl

#### Scenario: TextMissingWhen:Line
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A plain-text string that does not contain a 'When:' line
- **WHEN** parseOutlookText is called
- **THEN** null is returned

#### Scenario: OutlookTextWithMissingOptionalFields
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A string with a When: line but no Where:, Note:, or Invitees: lines
- **WHEN** parseOutlookText is called
- **THEN** A MeetingEvent is returned with empty location, organizer, and attendees, and description drawn from
remaining lines

#### Scenario: Date/timeLineMatchesExpectedPattern
- **GIVEN** The When: line contains a date/time in 'Monday, June 15, 2026 at 3:30 PM - 4:00 PM' format
- **WHEN** parseOutlookText is called
- **THEN** date is formatted as YYYY-MM-DD and startTime/endTime are extracted correctly

#### Scenario: Date/timeLineDoesNotMatchExpectedPattern
- **GIVEN** The When: line has an unrecognized date/time format
- **WHEN** parseOutlookText is called
- **THEN** date defaults to today's ISO date and startTime/endTime are empty strings

#### Scenario: DescriptionContainsTeamsBoilerplate
- **GIVEN** The free-form description block includes Microsoft Teams meeting boilerplate text
- **WHEN** parseOutlookText is called
- **THEN** The Teams boilerplate is stripped from the description field

#### Scenario: MeetingURLFoundInDescription
- **GIVEN** The description contains a recognized Teams/Zoom/Meet URL
- **WHEN** parseOutlookText is called
- **THEN** meetingUrl is populated from the description

#### Scenario: MeetingURLFoundInLocationOnly
- **GIVEN** The description has no meeting URL but the Where: line contains one
- **WHEN** parseOutlookText is called
- **THEN** meetingUrl is extracted from the location text

#### Scenario: MultiLineTitleBeforeWhen:
- **GIVEN** Multiple non-empty lines appear before the When: line
- **WHEN** parseOutlookText is called
- **THEN** All pre-When lines are joined with a space to form the title

#### Scenario: NoSubjectLinesBeforeWhen:
- **GIVEN** The When: line is the very first line of the text
- **WHEN** parseOutlookText is called
- **THEN** title defaults to 'Untitled Meeting'

### Requirement: StripTeamsBoilerplate

The system SHALL removes the microsoft teams meeting boilerplate block from a description string.

> Implementation: `stripTeamsBoilerplate` in `src/ts/icalParser.ts` · confidence: llm detects the block by a line of 8+ underscores followed by a localized 'microsoft teams meeting' header (english, french, spanish), or by the header appearing directly. returns the text up to (but not including) the boilerplate block.

#### Scenario: TextContainsTeamsBoilerplatePrecededByUnderscores
- **GIVEN** The description contains a line of 8+ underscores followed by a 'Microsoft Teams meeting' header
line
- **WHEN** stripTeamsBoilerplate is called
- **THEN** All content from the underscore separator onward is removed and trimmed text is returned

#### Scenario: TextContainsTeamsBoilerplateHeaderWithoutUnderscoreSeparator
- **GIVEN** The description contains a 'Microsoft Teams meeting' header line with no underscore separator before
it
- **WHEN** stripTeamsBoilerplate is called
- **THEN** All content from the header line onward is removed and trimmed text is returned

#### Scenario: FrenchLocalizedTeamsHeader
- **GIVEN** The description contains 'Réunion Microsoft Teams' as the boilerplate header
- **WHEN** stripTeamsBoilerplate is called
- **THEN** Content from the French header onward is removed

#### Scenario: SpanishLocalizedTeamsHeader
- **GIVEN** The description contains 'Reunión de Microsoft Teams' as the boilerplate header
- **WHEN** stripTeamsBoilerplate is called
- **THEN** Content from the Spanish header onward is removed

#### Scenario: TextContainsNoTeamsBoilerplate
- **GIVEN** The description has no underscore separators or Teams header lines
- **WHEN** stripTeamsBoilerplate is called
- **THEN** The original text is returned unchanged

#### Scenario: UnderscoreLineNotFollowedByTeamsHeader
- **GIVEN** The description contains a line of 8+ underscores but the following non-empty line is not a Teams
header
- **WHEN** stripTeamsBoilerplate is called
- **THEN** The text is returned unchanged (the separator is not treated as a boilerplate boundary)
