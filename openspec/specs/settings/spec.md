# Settings Specification

> Generated by openlore v1.0.0 on 2026-06-18 09:21
> Source files: src/ts/settingsTab.ts, src/ts/icalParser.ts, src/ts/ical.d.ts

## Purpose

Renders a drag-and-drop modal UI that lets the user import an .ics file either by dropping it onto
the zone or clicking to browse, and delegates the file/event payload to plugin-provided callbacks.

## Entities

### IcalMeetingNotesSettings

> `src/ts/settingsTab.ts`

Persisted configuration for the iCal Meeting Notes plugin, controlling where notes are saved, which
template is used, and how note sections are structured.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| notesFolder | string | Vault folder path where created meeting notes are saved. Empty string means vault root. |
| useActiveFolder | boolean | When true, new notes are saved in the folder of the currently open file, overriding notesFolder. |
| templateFile | string | Vault path to an Obsidian markdown template file. Empty string means use the built-in default template. |
| attendeesHeading | string | The exact markdown heading line under which attendee names are injected into the note. |
| notesHeading | string | The exact markdown heading line under which the meeting URL and description are injected into the note. |
| openAfterCreate | boolean | When true, the newly created meeting note is opened in the editor immediately after creation. |

**Relationships:**

- configures MeetingNote (Settings determine the save location, template, and section headings used when creating a MeetingNote.)
- references NoteTemplate (templateFile points to an optional vault file used as the note template.)

### MeetingEvent

> `src/ts/icalParser.ts`

A calendar meeting event parsed from either an ICS file or an Outlook for Mac plain-text clipboard
summary. Represents all the structured data needed to create a meeting note in Obsidian.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| title | string | The meeting subject/summary. Defaults to 'Untitled Meeting' when absent. |
| date | string | Meeting date in YYYY-MM-DD format. Defaults to today's date when absent. |
| startTime | string | Meeting start time in HH:mm (local) 24-hour format. Empty string when not parseable. |
| endTime | string | Meeting end time in HH:mm (local) 24-hour format. Empty string when not parseable. |
| organizer | string | Display name or email address of the meeting organizer. Empty string when not found. For ICS, organizer is excluded from attendees list. |
| attendees | string[] | Array of display names or email addresses of attendees, with the organizer excluded. Empty array when no attendees found. |
| description | string | Free-form meeting body/notes. Teams boilerplate sections are stripped before storage. |
| location | string | Physical or virtual location of the meeting. Empty string when not specified. |
| meetingUrl | string | Extracted video-conference join URL (Teams, Zoom, Google Meet, Webex). For ICS files the x-microsoft-skypeteamsmeetingurl property takes precedence over URLs found in description or location. Empty string when no URL is found. |

### CalendarComponent

> `src/ts/ical.d.ts`

Represents a parsed iCalendar component (e.g., VCALENDAR, VEVENT, VTIMEZONE). Acts as the structural
container for calendar data and its nested sub-components and properties.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| jcal | unknown | The raw jCal JSON structure passed to the constructor, representing the parsed iCal data |

**Relationships:**

- has-many CalendarComponent (A component can contain nested sub-components (e.g., VCALENDAR contains VEVENT))
- has-many CalendarProperty (A component owns one or more iCal properties such as SUMMARY, DTSTART, ATTENDEE)

### CalendarProperty

> `src/ts/ical.d.ts`

Represents a single iCalendar property (e.g., SUMMARY, DTSTART, ORGANIZER, ATTENDEE) including its
value and parameters such as CN or PARTSTAT.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| value | unknown | The typed value of the property, accessible via getFirstValue() |
| jsonRepresentation | [string, Record<string, string>, ...unknown[]] | Tuple of [propertyName, parametersMap, ...values] returned by toJSON() |

**Relationships:**

- belongs to CalendarComponent (A property is owned by and scoped to a parent CalendarComponent)

### CalendarTime

> `src/ts/ical.d.ts`

Represents a date or date-time value within an iCalendar data structure, such as DTSTART or DTEND of
an event. Bridges between iCal time representation and JavaScript Date.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| jsDate | Date | The equivalent JavaScript Date object, accessible via toJSDate() |

**Relationships:**

- belongs to CalendarProperty (CalendarTime is the resolved value of date/time-typed CalendarProperty instances such as DTSTART and DTEND)

### Requirement: MeetingEventValidation

The system SHALL validate MeetingEvent according to these rules:
- title defaults to 'Untitled Meeting' when the source has no subject/summary
- date defaults to today's ISO date (YYYY-MM-DD) when no date can be parsed
- startTime and endTime are empty strings (not null/undefined) when time cannot be parsed
- organizer must be excluded from the attendees array
- description must have Teams boilerplate stripped (content after an 8+ underscore divider followed by a Teams meeting header in EN/FR/ES, or directly after such a header)
- meetingUrl must match one of: teams.microsoft.com, *.zoom.us, meet.google.com, teams.live.com, webex.com, *.webex.com
- For ICS: x-microsoft-skypeteamsmeetingurl property takes priority over URLs extracted from description or location
- parseOutlookText returns null (not a MeetingEvent) when the input does not contain a 'When:' line
- parseIcs throws when the raw string is empty or contains no VEVENT component

#### Scenario: ParseICSWithFullEventData
- **GIVEN** A valid ICS string containing a VEVENT with summary, dtstart, dtend, organizer, attendees,
description, location, and x-microsoft-skypeteamsmeetingurl
- **WHEN** parseIcs() is called
- **THEN** A MeetingEvent is returned with all fields populated; organizer is resolved via CN parameter or
stripped mailto: address; organizer email is excluded from attendees; Teams URL comes from
x-microsoft-skypeteamsmeetingurl

#### Scenario: ParseICSWithMissingOptionalFields
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** An ICS string with a VEVENT that has no dtstart, dtend, organizer, attendees, description, or
location
- **WHEN** parseIcs() is called
- **THEN** A MeetingEvent is returned with title 'Untitled Meeting', date equal to today, startTime and endTime
as empty strings, organizer as empty string, attendees as empty array, and meetingUrl as empty
string

#### Scenario: ParseICSWithEmptyInput
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** An empty string or whitespace-only string
- **WHEN** parseIcs() is called
- **THEN** An error is thrown with message 'ICS file is empty.'

#### Scenario: ParseICSWithNoVEVENT
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A valid ICS string that contains no VEVENT component
- **WHEN** parseIcs() is called
- **THEN** An error is thrown with message 'No VEVENT found in the iCal file.'

#### Scenario: ParseOutlookPlainTextClipboard
- **GIVEN** A plain-text string in Outlook for Mac drag format with Subject, When:, Where:, Note:, and Invitees:
lines followed by a description
- **WHEN** parseOutlookText() is called
- **THEN** A MeetingEvent is returned with title from lines before 'When:', date/startTime/endTime parsed from
the When line, location from Where:, organizer from the Note: line pattern, and attendees split from
the Invitees: line

#### Scenario: RejectNonOutlookText
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A plain-text string that does not contain a 'When:' line
- **WHEN** parseOutlookText() is called
- **THEN** null is returned

#### Scenario: StripTeamsBoilerplateFromDescription
- **GIVEN** A MeetingEvent description containing a Microsoft Teams meeting section (preceded by 8+ underscores
or starting directly with the Teams header in EN, FR, or ES)
- **WHEN** parseIcs() or parseOutlookText() processes the description
- **THEN** The description field on the returned MeetingEvent contains only the content before the boilerplate
section

#### Scenario: ExtractMeetingURLFromDescriptionWhenNoDedicatedProperty
- **GIVEN** An ICS VEVENT with no x-microsoft-skypeteamsmeetingurl but a Teams/Zoom/Meet/Webex URL embedded in
the description or location
- **WHEN** parseIcs() is called
- **THEN** meetingUrl is populated with the first matching URL, with trailing >)] characters trimmed

### Requirement: Note name template setting field
The settings interface SHALL include a `noteNameTemplate` string field. The settings UI SHALL render a text input for this field with:
- Label and description (i18n)
- Placeholder text showing the default template (`{{date}} - {{title}}`) so users understand what an empty value means
- A read-only reference block listing all supported variables and their formats immediately below the input

#### Scenario: Setting persists across plugin reloads
- **WHEN** the user enters `{{date}} {{startTime}} - {{title}}` and saves
- **THEN** on next plugin load the field value is `{{date}} {{startTime}} - {{title}}`

#### Scenario: Empty value shows placeholder
- **WHEN** `noteNameTemplate` is `""` (default)
- **THEN** the text input shows the placeholder `{{date}} - {{title}}` (greyed out, not saved as a value)

### Requirement: Variable reference displayed in settings
The settings UI SHALL display a descriptive block beneath the note name template input that lists all available `{{placeholder}}` variables, their meaning, and the format of date and time values.

The block SHALL include:
- `{{date}}` — date in `YYYY-MM-DD` format (local timezone)
- `{{title}}` — meeting title
- `{{startTime}}` — start time in `HH:mm` 24-hour format (local timezone)
- `{{endTime}}` — end time in `HH:mm` 24-hour format (local timezone)
- `{{organizer}}` — organizer name or email (empty if not present)

#### Scenario: Reference block is always visible
- **WHEN** the settings tab is open
- **THEN** the variable reference block is visible below the note name template input regardless of the current field value
