# Note-creator Specification

> Generated by openlore v1.0.0 on 2026-06-18 09:21

## Purpose

Responsible for creating meeting note files in the Obsidian vault: resolving the target folder,
rendering a template with event data, injecting structured attendee and invite-notes blocks under
configured headings, and writing the resulting Markdown file.

## Requirements

> `src/ts/noteCreator.ts`

### Requirement: ResolveTargetFolder

The system SHALL determines the vault folder path where a new meeting note shall be saved.

> Implementation: `resolveTargetFolder` in `src/ts/noteCreator.ts` · confidence: llm when the 'useactivefolder' setting is enabled, it shall return the folder of the currently active file; otherwise it shall return the folder path from settings.

#### Scenario: ActiveFolderModeWithOpenFile
- **GIVEN** settings.useActiveFolder is true and a file is currently open in the workspace
- **WHEN** resolveTargetFolder is called
- **THEN** The parent folder path of the active file is returned

#### Scenario: ActiveFolderModeWithNoOpenFile
- **GIVEN** settings.useActiveFolder is true but no file is active in the workspace
- **WHEN** resolveTargetFolder is called
- **THEN** The configured settings.notesFolder is returned as fallback

#### Scenario: ActiveFolderModeWithFileAtVaultRoot
- **GIVEN** settings.useActiveFolder is true and the active file's parent path is '/'
- **WHEN** resolveTargetFolder is called
- **THEN** An empty string is returned, indicating the vault root

#### Scenario: FixedFolderMode
- **GIVEN** settings.useActiveFolder is false
- **WHEN** resolveTargetFolder is called
- **THEN** settings.notesFolder is returned regardless of the active file

### Requirement: CreateMeetingNote

The system SHALL creates a new markdown meeting note in the vault for the given event.

> Implementation: `t` in `src/ts/i18n.ts` · confidence: heuristic it shall sanitize the event title for use as a filename, load the configured template (or the built-in template), substitute {{date}} and {{title}} placeholders, inject the attendees list under the configured attendees heading, inject the meeting url / location / description block under the configured notes heading, ensure the target folder exists, and write the file to the vault. if a note with the same path already exists, it shall return the existing file without overwriting and display a notice to the user.

#### Scenario: SuccessfulNoteCreationWithBuiltInTemplate
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** No note exists at the computed path and settings.templateFile is empty
- **WHEN** createMeetingNote is called
- **THEN** A new Markdown file is created using the built-in template with date and title substituted

#### Scenario: SuccessfulNoteCreationWithCustomTemplate
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** No note exists at the computed path and settings.templateFile points to a valid vault file
- **WHEN** createMeetingNote is called
- **THEN** The custom template file is read, placeholders are substituted, and the note is created

#### Scenario: NoteAlreadyExists
- **GIVEN** A TFile already exists at the computed file path
- **WHEN** createMeetingNote is called
- **THEN** A user-facing Notice is shown and the existing TFile is returned without modification

#### Scenario: AttendeesBlockInjected
- **GIVEN** The event has an organizer and/or attendees
- **WHEN** createMeetingNote is called
- **THEN** A formatted attendee list is inserted under the heading defined by settings.attendeesHeading

#### Scenario: NotesBlockInjected
- **GIVEN** The event has a meetingUrl, location, or description
- **WHEN** createMeetingNote is called
- **THEN** Meeting URL, location, and description are inserted under the heading defined by
settings.notesHeading

#### Scenario: TargetFolderDoesNotExist
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** The resolved target folder path does not exist in the vault
- **WHEN** createMeetingNote is called
- **THEN** The folder is created before the note file is written

#### Scenario: TitleContainsFilesystemUnsafeCharacters
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** The event title contains characters such as / : * ? " < > | or \
- **WHEN** createMeetingNote is called
- **THEN** Those characters are replaced with hyphens in the filename, and the note is created successfully

#### Scenario: CustomTemplateFileNotFound
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** settings.templateFile is set but does not resolve to a TFile in the vault
- **WHEN** createMeetingNote is called
- **THEN** The built-in template is used as a fallback and the note is created successfully


## Sub-components

> `NoteCreatorService` is an orchestrator. Each sub-component below implements one logical block.

### Sub-component: ResolveTargetFolder

> Implements: `resolveTargetFolder`

Determines the vault-relative folder path where the new meeting note will be written, preferring the active file's parent folder when configured.

#### Requirement: ResolveActiveFolder

The system SHALL when useactivefolder is enabled, the function shall return the parent folder path of the currently active file in the workspace. if the active file sits at the vault root (parent path is '/' or absent), it must return an empty string.

#### Scenario: ActiveFileInSubfolder
- **GIVEN** useActiveFolder is true and the active file's parent path is 'daily/2026'
- **WHEN** resolveTargetFolder is called
- **THEN** the returned path is 'daily/2026'

#### Scenario: ActiveFileAtVaultRoot
- **GIVEN** useActiveFolder is true and the active file's parent path is '/'
- **WHEN** resolveTargetFolder is called
- **THEN** the returned path is an empty string

#### Scenario: NoActiveFileOpen
- **GIVEN** useActiveFolder is true and getActiveFile returns null
- **WHEN** resolveTargetFolder is called
- **THEN** the returned path falls through to settings.notesFolder

#### Requirement: FallbackToConfiguredFolder

The system SHALL when useactivefolder is disabled or no active file exists, the function shall return settings.notesfolder verbatim.

#### Scenario: UseActiveFolderDisabled
- **GIVEN** useActiveFolder is false and notesFolder is 'meetings'
- **WHEN** resolveTargetFolder is called
- **THEN** the returned path is 'meetings'

### Sub-component: NormalizePath

> Implements: `normalizePath`

Normalises raw path strings into vault-safe paths by resolving redundant separators and ensuring consistent slash direction, as required by the Obsidian API before any vault file operation.

#### Requirement: NormaliseFilePath

The system SHALL the call shall convert any raw concatenated path (folder + filename) into a canonical obsidian vault path, removing double slashes and standardising separators.

#### Scenario: PathWithDoubleSlashes
- **GIVEN** rawPath is 'meetings//2026-06-18 - Standup.md'
- **WHEN** normalizePath is called
- **THEN** the returned path is 'meetings/2026-06-18 - Standup.md'

#### Scenario: EmptyFolderPrefix
- **GIVEN** rawPath is '2026-06-18 - Standup.md' (no folder prefix)
- **WHEN** normalizePath is called
- **THEN** the returned path is '2026-06-18 - Standup.md'

### Sub-component: Translate

> Implements: `t`

Looks up a localised string by key and performs variable interpolation, replacing every {{varName}} placeholder with the corresponding value supplied in the vars map.

#### Requirement: KeyLookup

The system SHALL the function shall return the translated string for the given key from the active locale's string table, falling back to the english ('en') string when the active locale does not define that key.

#### Scenario: KeyPresentInActiveLocale
- **GIVEN** the locale is 'fr' and 'organizer_label' is defined in fr.json
- **WHEN** t('organizer_label') is called
- **THEN** the French string is returned

#### Scenario: KeyMissingFromActiveLocale
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** the locale is 'fr' but 'organizer_label' is absent from fr.json
- **WHEN** t('organizer_label') is called
- **THEN** the English fallback string from en.json is returned

#### Requirement: VariableInterpolation

The system SHALL when a vars map is provided, the function shall replace every occurrence of {{varname}} in the resolved string with its corresponding value using object.entries and str.replaceall.

#### Scenario: SingleVariableSubstitution
- **GIVEN** vars is { name: 'standup.md' } and the string template is 'Note {{name}} already exists'
- **WHEN** t('notice.note_exists', { name: 'standup.md' }) is called
- **THEN** the returned string is 'Note standup.md already exists'

#### Scenario: NoVarsProvided
- **GIVEN** vars is undefined
- **WHEN** t('organizer_label') is called
- **THEN** the raw looked-up string is returned unchanged

### Sub-component: TemplatePlaceholderReplace

> Implements: `raw.replace`

Performs a global search-and-replace on the raw template string, substituting every {{date}} and {{title}} placeholder with the corresponding event field values.

#### Requirement: ReplaceDatePlaceholder

The system SHALL the call shall replace all occurrences of the literal string '{{date}}' in the template with the event's date string using a global regex.

#### Scenario: DateAppearsMultipleTimes
- **GIVEN** the template contains '{{date}}' in both the frontmatter title field and the H1 heading
- **WHEN** the replace chain executes
- **THEN** both occurrences are replaced with the event date

#### Requirement: ReplaceTitlePlaceholder

The system SHALL the call shall replace all occurrences of the literal string '{{title}}' in the already date-substituted string with the event's title string using a second chained global regex replace.

#### Scenario: TitleWithSpecialCharacters
- **GIVEN** event.title is 'Q2 Planning: Budget & Roadmap'
- **WHEN** the replace chain executes
- **THEN** all {{title}} occurrences are replaced with 'Q2 Planning: Budget & Roadmap' without further escaping

#### Scenario: CustomTemplateHasNoPlaceholders
- **GIVEN** the template string contains no {{date}} or {{title}} tokens
- **WHEN** the replace chain executes
- **THEN** the content is returned unchanged

### Sub-component: BuildAttendeesBlock

> Implements: `buildAttendeesBlock`

Assembles a Markdown bullet-list string listing the organizer (labelled via i18n) followed by all attendees, to be injected under the attendees heading.

#### Requirement: PrependOrganizer

The system SHALL when the event has a non-empty organizer field, the function shall push a bullet item of the form '- {organizer} ({t(organizer_label)})' as the first element using items.push.

#### Scenario: OrganizerPresent
- **GIVEN** event.organizer is 'alice@example.com' and t('organizer_label') returns 'organizer'
- **WHEN** buildAttendeesBlock is called
- **THEN** the first bullet is '- alice@example.com (organizer)'

#### Scenario: NoOrganizer
- **GIVEN** event.organizer is undefined
- **WHEN** buildAttendeesBlock is called
- **THEN** no organizer bullet is prepended and items starts with attendees

#### Requirement: AppendAttendees

The system SHALL for each attendee in event.attendees, the function shall push a bullet item '- {attendee}' in order using items.push, then join all items with newlines.

#### Scenario: MultipleAttendees
- **GIVEN** event.attendees is ['bob@example.com', 'carol@example.com']
- **WHEN** buildAttendeesBlock is called
- **THEN** the block contains '- bob@example.com' and '- carol@example.com' separated by newlines

#### Scenario: EmptyAttendeesList
- **GIVEN** event.attendees is [] and event.organizer is undefined
- **WHEN** buildAttendeesBlock is called
- **THEN** the returned string is empty

### Sub-component: InjectUnderHeading

> Implements: `injectUnderHeading`

Finds a target heading line in Markdown content and inserts a block of text immediately below it, replacing a lone placeholder bullet if present or prepending a blank line otherwise.

#### Requirement: LocateHeading

The system SHALL the function shall split the content string into lines and use lines.findindex to locate the first line whose trimmed value equals the trimmed heading. if the heading is not found it must return the content unchanged.

#### Scenario: HeadingFound
- **GIVEN** content contains the line '## Attendees' and heading is '## Attendees'
- **WHEN** injectUnderHeading is called
- **THEN** idx points to that line and injection proceeds

#### Scenario: HeadingNotFound
- **GIVEN** content does not contain the heading string
- **WHEN** injectUnderHeading is called
- **THEN** the original content string is returned unmodified

#### Requirement: ReplacePlaceholderBullet

The system SHALL if the first non-empty line after the heading is a lone dash ('- ' or '-'), the function shall use lines.splice to replace that line with the block in-place.

#### Scenario: LoneDashPlaceholderPresent
- **GIVEN** the line immediately after the heading (ignoring blanks) is '-'
- **WHEN** injectUnderHeading is called with a non-empty block
- **THEN** the lone dash line is replaced by the block and no extra blank line is inserted

#### Requirement: InsertAfterHeading

The system SHALL when no lone-dash placeholder exists immediately after the heading, the function shall splice an empty line followed by the block after the heading line, preserving all subsequent content.

#### Scenario: ContentAlreadyFollowsHeading
- **GIVEN** the line after the heading is existing non-placeholder content
- **WHEN** injectUnderHeading is called
- **THEN** an empty line and then the block are inserted between the heading line and the existing content

#### Scenario: EmptyOrUndefinedHeadingOrBlock
- **GIVEN** heading is an empty string or block is an empty string
- **WHEN** injectUnderHeading is called
- **THEN** the content is returned unchanged without any modification

### Sub-component: BuildNotesBlock

> Implements: `buildNotesBlock`

Assembles a Markdown blockquote and prose string containing the meeting URL, physical location, and description, to be injected under the invite-notes heading.

#### Requirement: AppendMeetingUrl

The system SHALL when event.meetingurl is non-empty, the function shall push a blockquote line '> {meetingurl}' as the first part using parts.push.

#### Scenario: MeetingURLPresent
- **GIVEN** event.meetingUrl is 'https://meet.example.com/abc'
- **WHEN** buildNotesBlock is called
- **THEN** the first part is '> https://meet.example.com/abc'

#### Scenario: NoMeetingURL
- **GIVEN** event.meetingUrl is undefined
- **WHEN** buildNotesBlock is called
- **THEN** no URL blockquote line is added to parts

#### Requirement: AppendLocation

The system SHALL when event.location is non-empty and differs from event.meetingurl, the function shall push a blockquote location line '> 📍 {location}' using parts.push.

#### Scenario: DistinctPhysicalLocation
- **GIVEN** event.location is 'Room 42' and event.meetingUrl is 'https://meet.example.com/abc'
- **WHEN** buildNotesBlock is called
- **THEN** a line '> 📍 Room 42' is included in parts

#### Scenario: LocationEqualsMeetingURL
- **GIVEN** event.location and event.meetingUrl are identical strings
- **WHEN** buildNotesBlock is called
- **THEN** the location line is NOT added to avoid duplication

#### Requirement: AppendDescription

The system SHALL when event.description is non-empty, the function shall push an empty string separator (if other parts already exist) and then the description text using parts.push before joining all parts with newlines.

#### Scenario: DescriptionWithPrecedingURL
- **GIVEN** event.meetingUrl is set and event.description is 'Quarterly review agenda'
- **WHEN** buildNotesBlock is called
- **THEN** the block contains an empty line between the URL blockquote and the description text

#### Scenario: DescriptionOnly,NoURLOrLocation
- **GIVEN** event.meetingUrl and event.location are undefined, event.description is 'Agenda'
- **WHEN** buildNotesBlock is called
- **THEN** the block is simply 'Agenda' with no leading blank line or blockquotes

#### Scenario: AllFieldsEmpty
- **GIVEN** event.meetingUrl, event.location, and event.description are all undefined
- **WHEN** buildNotesBlock is called
- **THEN** the returned string is empty

### Sub-component: EnsureFolder

> Implements: `ensureFolder`

Guarantees the target folder exists in the Obsidian vault before the note file is written, creating it if absent and short-circuiting when the path is empty.

#### Requirement: SkipEmptyPath

The system SHALL when folderpath is an empty string (vault root), the function shall return immediately without performing any vault operation.

#### Scenario: EmptyFolderPath
- **GIVEN** folderPath is an empty string
- **WHEN** ensureFolder is called
- **THEN** no vault API call is made and the function resolves

#### Requirement: SkipExistingFolder

The system SHALL when app.vault.getabstractfilebypath returns a tfolder instance for the given path, the function shall return without calling createfolder.

#### Scenario: FolderAlreadyExists
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** getAbstractFileByPath('meetings') returns a TFolder
- **WHEN** ensureFolder is called with 'meetings'
- **THEN** createFolder is NOT called

#### Requirement: CreateMissingFolder

The system SHALL when the path is non-empty and getabstractfilebypath does not return a tfolder, the function shall call app.vault.createfolder(folderpath) and await its completion.

#### Scenario: FolderDoesNotExist
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** getAbstractFileByPath('new-folder') returns null
- **WHEN** ensureFolder is called with 'new-folder'
- **THEN** app.vault.createFolder is called with 'new-folder' and awaited
